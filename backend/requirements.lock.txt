# Pinned lock file for reproducible builds (P47).

# `requirements.txt` uses `~=` to cap minor versions (prevents accidental
# upgrades across major versions while allowing patch updates).
# This lock file pins every transitive dependency to an exact version + hash,
# matching the known-good environment that last passed `pytest 780+ passed`.
#
# How to regenerate (in an environment with pip + network access):
#
#   cd backend
#   pip-compile requirements.in --generate-hashes --output-file=requirements.lock.txt
#   # or, if pip-compile unavailable:
#   rm -rf .venv && python3 -m venv .venv && source .venv/bin/activate
#   pip install -r requirements.txt -r requirements-dev.txt
#   pip freeze --all | sort > requirements.lock.txt
#
# How to install from the lock file:
#
#   pip install --require-hashes -r requirements.lock.txt
#
