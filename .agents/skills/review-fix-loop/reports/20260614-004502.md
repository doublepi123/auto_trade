**Fix 1.1** — `backend/app/api/credentials.py`（L67-95）  
凭证更新失败时不再把 `str(exc)` 原样写入审计日志。改为 `{"error": type(exc).__name__, "detail": str(exc)[:256]}`，避免任意 webhook URL/token 落到 audit_logs。

**Fix 1.2** — `backend/app/core/audit.py` `extract_ip`（L77-92）  
不再信任 `X-Forwarded-For` 头。当前 docker-compose 后端无反代，XFF 头是攻击者可控的。改用 `request.client.host` 真实 socket peer。

**Fix 1.3** — `backend/app/api/ws.py`（L73-122）  
为 post-accept auth 帧加 `_WS_AUTH_MAX_FRAME_BYTES = 4096` 长度检查，超过则拒绝。修复一个慢客户端/攻击者在 5s auth 窗口内上传任意大小 text frame 导致内存耗尽的问题。

**Fix 1.4** — `backend/app/services/trade_execution_service.py` `_coerce_order_status`（L1316-1340）  
broker 端 reconcile 路径下，broker 报告 status 但缺 `executed_quantity` 时不再硬编码 `Decimal('0')`，改回 `None`。runner 的 `_update_order_status` 已有 `if x is not None: order.x = x` 守卫，传 0 会**覆盖**之前已记录的部分成交值，导致 PnL 计算漏单。  
验证：49 个 trade_execution_service 测试 + 6 个 runner 测试全部通过。

**Fix 1.5** — `frontend/src/composables/useStatusStream.ts` `reconnectNow()`（L126-140）  
重置 `wsAuthRejected = false` 与 `reconnectAttempts = 0`，并清理 reconnectTimer。一次 1008（凭据过期等）后用户之前会永远卡在 polling，现在能恢复 WS。

**Fix 1.6** — `frontend/src/composables/useNotificationStream.ts` `_enable()`（L239-256）  
改为 `async`，先 `await backfill()` 再 `setInterval(poll)`，避免首轮 poll 跑得比 backfill 快、把 backfill 已经标记为 known 的旧事件当新事件再推一次。

**Fix 1.7** — `backend/tests/test_database.py` `test_sqlite_wal_and_busy_timeout_enabled`（L464-496）  
原测试访问 `database.engine` 而不 monkeypatch，在 host dev DB 上跑会污染。改为先设 `AUTO_TRADE_DATABASE_URL` 到 `tmp_path` 再 `importlib.reload(database)`，并修掉 `row[1]` 的 IndexError（`PRAGMA journal_mode` 返回单值 1-tuple，无列名）。

**Fix 1.8** — `backend/tests/test_audit_logger.py`（L70-83, L100-114）  
更新两个 `test_extract_ip_prefers_x_forwarded_for` / `test_extract_actor_returns_hash_and_ip` 用例反映 XFF 不再被信任的策略，保留断言 socket peer（`127.0.0.1`）而不是 XFF 值。注释解释原因。
