## 变更摘要

### 1. `backend/app/main.py` — 竞态条件修复 (HIGH)
`_llm_analysis_cron` 每 60 秒调用 `_llm_analysis_tick()`，后者通过 `await asyncio.to_thread(...)` 释放事件循环。若前次 tick 耗时超过 60 秒（LLM 调用慢时可能发生），两个 tick 将并发执行，导致：
- 全局状态竞态（`_last_llm_trigger_price`, `_llm_analysis_timestamps` 等）
- 可能重复发送 LLM 订单决策

修复：添加 `asyncio.Lock()` 确保同一时刻只有一个 tick 运行。

### 2. `backend/app/config.py` + `backend/tests/test_config.py` — 默认模型对齐 (MEDIUM)
CLAUDE.md 声明 LLM 模型默认为 `deepseek-v4-pro`，但代码实际默认为 `deepseek-v4-flash`。改为 `deepseek-v4-pro` 对齐项目文档与代码。

### 3. `backend/app/api/trade.py` — 事务处理改进 (LOW)
`_record_control_trace` 在 `finally` 块中无条件调用 `db.rollback()`，虽在 `commit()` 后为 no-op，但模式不规范。改为仅在 `except` 块中 rollback。

### 4. `backend/app/database.py` — SQL 注入风险消除 (LOW)
`_ensure_credential_config_notification_channels_column` 拼接字符串构造 ALTER TABLE 的 DEFAULT 子句。改用 `DEFAULT ''` + 参数化 UPDATE 回填。

## 最终建议
- 前端 `useNotificationStream.ts` 中 `playNotificationSound` 每次创建新 `AudioContext` 未关闭，频繁通知时可能积累资源。建议复用 AudioContext 或调用 `ctx.close()`。
- `trade.py` 中 `_is_today` 使用系统本地时区而非市场时区，对 HK 市场在 UTC 服务器上可能偏移一天，但仅影响前端展示过滤，不涉及交易逻辑。
- 建议为 `_llm_analysis_tick` 添加超时保护（当前无超时机制，若 LLM 调用永久挂起，将永久阻塞后续 tick）。
