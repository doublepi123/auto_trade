## 关键用户指出的 P1/P2/P3（迭代 12 重点处理）

1. **P1 longport 装进 lock 文件** ✅
   - lock 文件原本只 `fastapi/sqlalchemy/...` 不含 longport，Docker 镜像会缺券商 SDK
   - 修复：`requirements.lock.txt` 加 `longport==3.0.23`（实际版本），同步 `requirements.in` 和 `requirements.txt` 的 `~=` 上限

2. **P1 runner 双重 reconcile 循环** ✅
   - `_drive_reconcile_per_pending` 在 runner 层逐个 pending 调 `reconcile()`，但 `reconcile()` 内部已经遍历全部 pending；多 symbol 时 A 的 engine snapshot 会被错误 restore 到 B 的 runtime
   - 修复：去掉外层循环；每个 `_PendingOrder` 入轨时已绑自己的 `restore_engine_snapshot_fn`，runner 层只提供 `_fallback_restore` 兜底

3. **P2 LLM preview throttle 时间戳** ✅
   - `analyze()` 路径在迭代 8 已修（移到 `_call_llm` 成功后），但 `preview()` 路径 line 511 仍在 throttle 检查后立即更新
   - 修复：preview 路径同样把 timestamp 更新移到 `_call_llm` + `_parse_response` 成功之后

4. **P3 三个前端导出 link.remove() 缺失** ✅
   - DecisionTimeline.vue / Reports.vue / Review.vue 都缺 `link.remove()`，每次导出累积 hidden `<a>` 节点
   - 修复：三处都补上 `if (link.parentNode) link.parentNode.removeChild(link)`
